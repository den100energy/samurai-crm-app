# Система рассрочки платежей — Спецификация

## Executive Summary
Система учёта рассрочки/отсрочки платежей за абонементы для школы боевых искусств "Самурай". Позволяет оформлять гибкие графики платежей (20% аванс + остаток), отслеживать погашения, автоматически напоминать клиентам и блокировать доступ при просрочке.

## Problem Statement
**Текущая проблема:** Клиенты покупают длинные абонементы (6-7 месяцев) с рассрочкой и тянут оплату до самого конца, получая выгоду от большого абонемента без реальной оплаты. Учёт ведётся в Excel/бумаге, что приводит к потере информации и несвоевременным напоминаниям.

**Цель:** Автоматизировать учёт графиков рассрочки, напоминания и контроль оплаты в CRM.

## Success Criteria
- Администратор может оформить рассрочку при создании абонемента за < 2 минуты
- Все просроченные платежи видны в одном месте
- Клиенты получают напоминания за 3 дня до платежа
- При просрочке > 5 дней ученик автоматически блокируется
- Формируется PDF-приложение к договору об отсрочке

## User Personas

### Администратор/Владелец
- Оформляет рассрочки
- Отслеживает должников
- Фиксирует поступление платежей
- Генерирует документы

### Клиент (ученик/родитель)
- Видит свой график платежей в личном кабинете
- Получает напоминания
- Понимает последствия просрочки

### Бухгалтер
- Видит отчёт по дебиторке
- Анализирует поступления по рассрочкам

## User Journey

### Оформление рассрочки (Администратор)
1. При создании абонемента ставит галочку "Рассрочка"
2. Открывается форма:
   - Первоначальный взнос: ___ ₽ (default: 20% от суммы)
   - Кнопка "+ Добавить платёж" для создания графика
   - Для каждого платежа: сумма + дата
3. Система проверяет: сумма всех платежей = стоимость абонемента
4. Сохраняет абонемент + график рассрочки
5. Генерирует PDF-приложение к договору (по запросу)

### Просмотр должников (Администратор)
1. Заходит на страницу /installments
2. Видит виджеты:
   - Просроченные (красный, сортировка по дням просрочки)
   - На этой неделе (жёлтый)
   - Общая дебиторка (сумма)
3. Таблица всех активных рассрочек с прогресс-барами
4. Может кликнуть на клиента → карточка с деталями

### Фиксация платежа
1. В карточке рассрочки нажимает "Оплачено" напротив платежа
2. Вводит фактическую сумму и дату
3. Платёж отмечается как оплаченный
4. Автоматически создаётся запись в payments (категория "Доплата за абон.")

### Просмотр графика (Клиент в кабинете)
1. Открывает личный кабинет
2. В блоке "Абонемент" видит:
   - "Рассрочка: 3 600 / 18 000 ₽ оплачено"
   - График платежей с датами и статусами
3. При приближении платежа — предупреждение
4. При просрочке — красное уведомление с суммой долга

## Technical Architecture

### Data Model

```sql
-- Планы рассрочки
CREATE TABLE installment_plans (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  subscription_id UUID REFERENCES subscriptions(id) ON DELETE CASCADE,
  total_amount NUMERIC NOT NULL,
  deposit_amount NUMERIC NOT NULL DEFAULT 0,
  deposit_paid_at DATE,
  status TEXT DEFAULT 'active' CHECK (status IN ('active', 'completed', 'cancelled')),
  reminder_days INT DEFAULT 3,
  grace_period_days INT DEFAULT 5,
  notes TEXT,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- Платежи по рассрочке
CREATE TABLE installment_payments (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  plan_id UUID REFERENCES installment_plans(id) ON DELETE CASCADE,
  amount NUMERIC NOT NULL,
  due_date DATE NOT NULL,
  status TEXT DEFAULT 'pending' CHECK (status IN ('pending', 'paid', 'overdue')),
  paid_at DATE,
  actual_amount NUMERIC,
  payment_id UUID REFERENCES payments(id),
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- Индексы
CREATE INDEX idx_installment_plans_subscription ON installment_plans(subscription_id);
CREATE INDEX idx_installment_plans_status ON installment_plans(status);
CREATE INDEX idx_installment_payments_plan ON installment_payments(plan_id);
CREATE INDEX idx_installment_payments_due_date ON installment_payments(due_date);
CREATE INDEX idx_installment_payments_status ON installment_payments(status);
```

### System Components

1. **InstallmentForm** — компонент формы создания рассрочки
2. **InstallmentDashboard** — страница /installments
3. **InstallmentCard** — карточка рассрочки с платежами
4. **InstallmentClientView** — блок в кабинете клиента
5. **installment-cron** — Vercel Cron для обновления статусов и напоминаний
6. **InstallmentPDF** — генератор PDF документа

### Integrations

- **payments** — при фиксации платежа создаётся запись в payments
- **subscriptions** — рассрочка привязана к абонементу
- **students** — статус blocked при просрочке > grace_period
- **Уведомления** — интеграция с существующей системой (частично)

### Security Model
- Администратор: полный доступ к CRUD операциям
- Клиент: только чтение своей рассрочки через токен кабинета
- Тренер: только просмотр статуса оплаты (для контроля допуска)

## Non-Functional Requirements
- **Performance**: Страница дашборда загружается < 2 сек
- **Reliability**: Cron-задача выполняется ежедневно в 8:00 (напоминания) и 00:00 (статусы)
- **Data Integrity**: Сумма платежей = total_amount - deposit_amount

## Out of Scope
- Автоматическое списание с карты
- Интеграция с банком/эквайрингом для автосверки
- Пени и штрафы за просрочку (только блокировка)
- Юридическая работа с должниками

## Appendix: Research Findings

### Опыт других сервисов

**Фитнес-клубы (WorldClass, DDX Fitness):**
- Рассрочка обычно 3-6 месяцев
- Автоматическое списание с карты
- Блокировка при неуспешном списании

**Образовательные платформы (Skillbox, Geekbrains):**
- Интеграция с банковской рассрочкой (Тинькофф, Сбер)
- Гибкий график: 6/12/24 месяца
- Напоминания за 3 дня по email + push

**SaaS-сервисы:**
- Net-30/Net-60 условия для B2B
- Автоматические напоминания через email
- Блокировка функционала при просрочке

### Рекомендации для "Самурай"
- **Гибкий график** — правильный выбор при 5-15 клиентах
- **Grace period 5 дней** — достаточно лояльно для школы
- **Напоминания за 3 дня** — оптимально
- **PDF-документ** — важно для юридической защиты
