# Методология: Срезы прогресса и AI-программы развития
## Samurai CRM — документ для доработки системы

---

## 1. Контекст школы

**Школа Самурая** — клуб айкидо айкикай. Основной акцент:
- Принципы ненасилия, работа с балансом
- Партнёрское взаимодействие (не соревновательный вид)
- Группы: Старт (дети 4–9 / младший возраст), Основная нач., Основная оп., Цигун, Индивидуальные
- Тренировки 2–3 раза в неделю, абонементы на 8/12/16/36 занятий

---

## 2. Жизненный путь ученика в системе

```
Лид → Пробное занятие → Анкета 1 (диагностика)
                              ↓
                    Тренер заполняет оценки с пробного
                              ↓
                    AI генерирует Программу на 1 месяц
                              ↓
                    Через ~28 дней → Срез прогресса (Анкета 2)
                    Заполняют: родитель + тренер
                              ↓
                    AI сравнивает Анкету 1 и Срез 1 → новая программа
                              ↓
                    Повторные срезы (раз в ~месяц по решению тренера)
```

---

## 3. Анкета 1 — Начальная диагностика новичка

**Кто заполняет:** родитель (онлайн, по ссылке) + тренер (в CRM после пробного)

**Поля которые заполняет родитель:**
- Имя, возраст ученика
- Травмы / перенесённые операции
- Противопоказания к нагрузкам
- Другие секции / кружки сейчас
- Предыдущий спорт (что, как долго)
- Особенности характера ребёнка
- Ожидания семьи: "Чем хотите, чтобы мы помогли вашему ребёнку?"

**Оценки родителя — 16 качеств (шкала 1–10):**
| Код | Качество |
|-----|----------|
| strength | Сила |
| speed | Быстрота |
| endurance | Выносливость |
| agility | Ловкость |
| coordination | Координация |
| posture | Осанка |
| flexibility | Гибкость |
| discipline | Дисциплина |
| sociability | Общительность |
| confidence | Уверенность |
| learnability | Обучаемость |
| attentiveness | Внимательность |
| emotional_balance | Уравновешенность |
| goal_orientation | Целеустремлённость |
| activity | Активность |
| self_defense | Самозащита |

**Оценки тренера (те же 16 качеств, prefix `trainer_`):**
Тренер оценивает с пробного занятия — своим взглядом, независимо от родителя.

**Заметки тренера (`trainer_notes`):** свободный текст — наблюдения с пробного.

## 4. AI-программа на первый месяц (после Анкеты 1)

**API:** `POST /api/generate-program`

**Что передаётся в промпт:**
- Имя, возраст ученика
- Травмы, противопоказания, другие секции, предыдущий спорт
- Особенности характера
- Ожидания семьи
- 16 качеств: оценка родителя + оценка тренера (по каждому)
- Заметки тренера с пробного

**Текущая структура ответа AI:**
```
**Анализ ученика**
(2-3 предложения — кто перед нами, что выделяется)

**Сильные стороны**
(2-3 пункта — на что опираемся)

**Зоны развития**
(2-3 приоритетных направления с объяснением)

**Программа на 1 месяц**
Неделя 1: (фокус и задачи)
Неделя 2: (фокус и задачи)
Неделя 3: (фокус и задачи)
Неделя 4: (фокус и задачи)

**Рекомендации для родителей**
(3-4 конкретных совета — что делать дома)
```

**Модель:** `anthropic/claude-3-haiku` через OpenRouter  
**Max tokens:** 1500

## 5. Срез прогресса (Анкета 2 и повторные)

**Когда запускается:**
- Автоматически через ~28 дней после первой тренировки (cron)
- Вручную тренером из карточки ученика (кнопка "+ Новый срез")

**Кто заполняет:** родитель + тренер (независимо)

**Что заполняет родитель:**
- Те же 16 качеств (оценка `q_*`) — как видит сам

**Что заполняет тренер (в CRM):**
- Те же 16 качеств (оценка `trainer_*`) — по наблюдениям на тренировках
- Заметки тренера (`trainer_notes`) — свободный текст

**Поля среза в БД (`progress_surveys`):**
- `survey_number` — номер среза (1, 2, 3...)
- `title` — название (напр. "Рост — Апрель 2026")
- `filled_at` — когда заполнен
- `trainer_filled_at` — когда тренер внёс оценки
- `initiated_by` — кто запустил (auto / trainer)
- `parent_name`, `parent_phone` — кто заполнял со стороны родителей
- `filled_by` — 'parent' | 'student'

## 6. AI-анализ прогресса (сравнение срезов)

**API:** `POST /api/compare-surveys`

**Что передаётся:**
- `survey1` — данные Анкеты 1 (начальная диагностика, может отсутствовать)
- `surveys[]` — все заполненные срезы, отсортированные по дате (от старого к новому)
- `studentName` — имя ученика

**Логика построения промпта:**

**Случай A — есть Анкета 1 + 1 срез:**
- Блок "Диагностика → Первый срез": динамика по 16 качествам (родитель + тренер)
- Формат: `Качество: было → стало ↑+N`

**Случай B — есть Анкета 1 + 2+ среза:**
- Блок "Диагностика → Первый срез" (исходная база)
- Блок "Недавняя динамика": предпоследний → последний срез
- Блок "Общая динамика": первый срез → последний

**Случай C — нет Анкеты 1 (только срезы):**
- Снимок текущего среза (абсолютные цифры без сравнения)

**Текущая структура ответа AI:**
```
**Что изменилось за первый месяц** (или "в последнее время")
(3-4 конкретных наблюдения с цифрами)

**Общий путь за всё время** (если срезов ≥ 2)
(2-3 наблюдения о долгосрочной динамике)

**Главные достижения**
(2-3 пункта)

**На что обратить внимание**
(2-3 зоны для работы)

**Программа на следующий месяц**
Неделя 1: ...
Неделя 2: ...
Неделя 3: ...
Неделя 4: ...

**Слова для родителей**
(2-3 тёплых предложения)
```

**Модель:** `anthropic/claude-3-haiku` через OpenRouter  
**Max tokens:** 1200

## 7. Что сейчас не хватает / проблемы

1. **Программа слишком общая** — AI не знает конкретных упражнений айкидо, техник, уровня группы
2. **Нет контекста группы** — ученик из "Старт" (7 лет) и из "Основная (оп.)" получают одинаково сформулированные программы
3. **Недели слишком абстрактны** — "фокус на координации" без конкретики что делать на тренировке
4. **Нет разделения** — программа для родителей и программа для тренера — это разные документы с разным языком
5. **Заметки тренера слабо используются** — они есть в промпте, но AI не всегда на них опирается
6. **Нет домашних заданий** — конкретных упражнений которые можно делать дома 5-10 минут
7. **Нет учёта абонемента** — не известно сколько занятий в неделю у ученика (8 занятий ≠ 36 занятий)

## 8. Данные доступные но не используемые

Эти данные есть в системе, но не передаются в AI:

- **Группа ученика** (`group_name`) — Старт / Основная нач. / Основная оп.
- **Дата начала занятий** (`enrollment_date`, `training_start_date`)
- **Количество посещений** — сколько занятий уже посетил
- **Тип абонемента** — 8, 12, 16, 36 занятий (≈ частота тренировок)
- **История абонементов** — как долго занимается в школе
- **Аттестации / пояса** (`belts`) — достижения, уровень
- **CRM-статус** (`crm_status`) — active / at_risk / churned

## 9. Идеи для улучшения (для обсуждения)

### Структура выходного документа

Разделить на два документа:

**A. Карточка тренера** (внутренний документ):
- Технический фокус на ближайший месяц
- Конкретные техники айкидо для отработки
- Что наблюдать на тренировках
- Как работать с характером этого ученика

**B. Письмо родителям** (отправляется в Telegram):
- Тёплый тон, без жаргона
- Что ребёнок уже умеет
- На что обратить внимание дома
- Простые домашние упражнения (5-10 мин в день)
- Что ожидать в следующем месяце

### Контекст для промпта

Добавить в промпт:
- Группа ученика + возраст
- Стаж занятий (месяцев в школе)
- Количество посещённых тренировок
- Последний пояс/уровень если есть
- Частота тренировок (из абонемента)

### System prompt

Вынести в отдельный system prompt базу знаний:
- Программа обучения по уровням (что изучают в группе "Старт", что в "Основная")
- Ключевые техники айкидо по уровням
- Принципы школы и ценности
- Типичные упражнения для домашней практики
