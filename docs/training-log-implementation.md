# Журнал тренировки — Внедрение (Training Log Feature)

## Контекст
Тренеры после каждого занятия должны фиксировать, что было на тренировке: какие техники, элементы ОФП, растяжку, приёмы разбирали. Это нужно для:
- Отслеживания соответствия тематическому плану (Programma 1/2, НСАР)
- Избегания «белых зон» в подготовке (что-то забыли/пропустили)
- Истории прогресса группы

**Главное требование:** заполнение ≤ 5 минут, минимум ввода текста — чекбоксы.

---

## Что реализовано

### 1. База данных — таблица `training_logs`
**Файл:** `training-logs-migration.sql`

Новая таблица в Supabase для хранения записей о тренировках:
- Привязка к `group_name`, `trainer_name`, `date`
- Массивы чекбоксов для каждой категории техник
- Свободное поле `notes` для заметок тренера
- Индексы для быстрого поиска по дате/группе/тренеру
- Row Level Security (полный доступ всем — как в текущих таблицах проекта)

### 2. Библиотека техник
**Файл:** `src/lib/training-checklists.ts`

~100 элементов для чекбоксов, разделённых на две программы:

#### Ушу (из Programma 1.pdf, Programma 2.pdf):
| Категория | Кол-во | Примеры |
|-----------|--------|---------|
| Разминка | 6 | Бег, прыжки, игровая, суставная, дыхательная |
| Растяжка | 5 | Шпагаты, мостик, махи ногами |
| ОФП | 13 | Отжимания (кулаки/пальцы/стойка), пресс, подтягивания, мабу-статика, сколопендра, акробатика |
| Базовая техника | 30+ | Стойки (мабу, гунбу, сюйбу...), удары руками (чунцюань, туйчжан...), удары ногами (чжэнтитуй, таньтуй...), прыжки, подсечки |
| Прикладная техника | 12 | Защита от ударов, броски, циньна, саньда, коридор, набивка |
| Таолу | 20 | Комплексы групп 1-3 + с оружием |
| Цигун | 4 | Ицзиньцзин, бадуаньцзин, дыхание, медитация |

#### Айкидо (из образовательной программы НСАР):
| Категория | Кол-во | Примеры |
|-----------|--------|---------|
| Укэми (страховки) | 4 | Вперёд, назад, вбок, кайтэн |
| Перемещения | 6 | Сикко, танкен, хэнко, тай-сабаки, тэнкан, ирими |
| Техники | 14 | Иккё, никкё, санкё, йонкё, гокё, ирими-нагэ, тенчи-нагэ, кото-гаэси,сихо-нагэ, рандори |
| Работа с оружием | 6 | Боккен, дзё, танто-дори |

Каждый элемент имеет: `id`, `label`, `icon` (эмодзи), `group` (для группировки).

Экспортируемые типы: `TrainingLogData`, `WUSHU_SECTIONS`, `AIKIDO_SECTIONS`, `emptyTrainingLogData()`.

### 3. UI компонент — Bottom Sheet с чекбоксами
**Файл:** `src/components/TrainingLogSheet.tsx`

**Props:**
- `isOpen` / `onClose` — управление видимостью
- `groupName` / `trainerName` / `date` — контекст тренировки
- `existingLog` — если есть сохранённая запись, подгружает её

**Функционал:**
- Выдвижная панель снизу (bottom sheet) с анимацией slide-up
- Переключатель **🥋 Ушу** / **⚔️ Айкидо** в шапке
- Раскрывающиеся секции-аккордеон (по умолчанию первая открыта)
- Чекбоксы-кнопки в виде тегов — быстрый тап, сгруппированы по подкатегориям
- Счётчик отмеченных элементов (2/13) с цветовой индикацией (синий/зелёный)
- Поле для заметок тренера (textarea, необязательное)
- Кнопка «Сохранить журнал» или «Обновить журнал» (если редактирование)
- Свайп вниз для закрытия (мобильный)
- Адаптивность под тёмную/светлую тему

**Логика сохранения:**
- Если `existingLog` есть → `UPDATE` по ID
- Если нет → `INSERT` новой записи
- После сохранения → автозакрытие через 800мс

## UX-поток тренера

```
1. Тренер открывает /attendance или /trainer/attendance
2. Выбирает группу и дату
3. Отмечает посещаемость → «Сохранить посещаемость»
4. Появляется кнопка «📝 Добавить журнал тренировки»
5. Клик → открывается bottom sheet
6. Переключатель Ушу/Айкидо (автоопределяется по группе)
7. Раскрывает секции → тапает чекбоксы (3-5 минут)
8. Опционально — пишет заметку
9. «Сохранить журнал» → закрытие, кнопка меняется на «📝 Редактировать журнал»
```
